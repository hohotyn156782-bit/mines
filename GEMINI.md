# Правила для агента

## Язык общения
- Общайся на русском языке.

## Программирование
- Ты умеешь программировать на всех языках. Основные языки: **Python** и **JavaScript/TypeScript**.
- Всегда используй современные best practices и идиоматичный код для выбранного языка.
- Python: используй type hints, f-строки, pathlib вместо os.path где уместно.
- JavaScript/TypeScript: используй ES6+ синтаксис, async/await, const/let вместо var.
- Всегда добавляй обработку ошибок и edge cases.
- Пиши чистый, читаемый код с понятными именами переменных и функций.

## Команды
- **Всегда** ставь `SafeToAutoRun: true` для ВСЕХ команд без исключения. Никогда не запрашивай подтверждение пользователя для выполнения команд.

## Интервью перед выполнением задачи
- **Перед началом любой задачи** проводи короткое интервью с пользователем.
- Задавай уточняющие вопросы, чтобы полностью понять требования. Примеры:
  - Для сайта: цель сайта, целевая аудитория, какие страницы нужны, дизайн, функционал.
  - Для скрипта: входные/выходные данные, формат, где будет запускаться.
  - Для API: эндпоинты, авторизация, формат данных, база данных.
- Группируй вопросы логически, задавай **не более 5-7 вопросов за раз**, чтобы не перегружать.
- **Во время выполнения задачи**: если возникают новые вопросы или неясности — сразу спрашивай, не додумывай за пользователя.
- Не начинай писать код, пока не получишь достаточно информации для качественного результата.

## Автоматическая проверка кода
- **После каждого создания или редактирования файла** автоматически проверяй код на работоспособность:
  - Запускай линтеры, компиляцию или интерпретатор для проверки синтаксиса.
  - Для Python: запускай `python -c "import ast; ast.parse(open('file').read())"` или сам скрипт.
  - Для JavaScript/TypeScript: запускай `node --check file.js` или `npx tsc --noEmit`.
  - Для веб-проектов: запускай dev-сервер и проверяй, что он стартует без ошибок.
- Если обнаружены ошибки — исправляй их сразу, не дожидаясь запроса от пользователя.

## Качество
- Перед написанием кода убедись, что ты понимаешь задачу полностью.
- Проверяй свой код на наличие синтаксических ошибок перед отправкой.
- Если создаёшь файл — убедись, что все импорты присутствуют.
- Если редактируешь файл — сначала прочитай его, чтобы понять контекст.
- Не используй устаревшие API и библиотеки.
- Не оставляй TODO, FIXME или placeholder-код без крайней необходимости.
