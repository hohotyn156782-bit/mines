## 3. CSS и визуал

**Статус:** ✅ Все проблемы исправлены

### Исправленные проблемы

#### Проблема — Неиспользуемые CSS-стили корзины (~150 строк)
- **Где:** styles.css
- **Серьёзность:** предупреждение
- **Что было не так:** Стили для `.cart-btn`, `.cart-count`, `.cart-items`, `.cart-item`, `.cart-footer`, `.modal`, `.close-modal`, `.add-to-cart-btn` — ни один класс не используется
- **Что сделано:** Удалены все ~150 строк мёртвого CSS

#### Проблема — Неиспользуемые CSS-стили toast
- **Где:** styles.css
- **Серьёзность:** предупреждение
- **Что было не так:** `.toast-container`, `.toast`, анимации `toastIn`/`toastOut` — не используются
- **Что сделано:** Удалены

#### Проблема — cursor: none в CSS без JS-fallback
- **Где:** styles.css
- **Серьёзность:** критично
- **Что было не так:** Системный курсор скрыт через CSS — если JS не загрузится, пользователь потеряет курсор
- **Что сделано:** Убран `cursor: none` из CSS, перенесён в JS с проверкой `pointer: fine` (тачскрины не затрагиваются)

#### Проблема — background-attachment: fixed на мобильных
- **Где:** styles.css
- **Серьёзность:** предупреждение
- **Что было не так:** На iOS `background-attachment: fixed` не работает и вызывает подёргивания
- **Что сделано:** Добавлен медиа-запрос `@media (max-width: 768px)` с `background-attachment: scroll`

- **Что сделано:** Удалена

#### Проблема — Кастомный курсор не работает на динамических карточках
- **Что было не так:** Обработчики mouseenter/mouseleave не обновлялись при перерисовке меню
- **Что сделано:** Заменены на делегирование событий через document

#### Проблема — cursor:none без проверки устройства
- **Что сделано:** Добавлена проверка `pointer: fine`, cursor:none устанавливается через JS

#### Проблема — animateRing работает бесконечно
- **Где:** script.js
- **Серьёзность:** инфо
- **Что было не так:** requestAnimationFrame вызывался постоянно, даже когда мышь не двигается
- **Что сделано:** Добавлен флаг `ringAnimating` — анимация запускается только при движении мыши и останавливается, когда кольцо догоняет курсор

#### Проблема — feather.replace() вызывается в 5 местах
- **Где:** script.js
- **Серьёзность:** инфо
- **Что не так:** Каждый вызов сканирует DOM. Однако каждый вызов необходим — срабатывает после конкретных DOM-изменений (переключение меню, рендер карточек, отправка формы). Не является дубликатом
